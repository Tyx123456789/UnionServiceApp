# UnionServiceApp Google Cloud 部署指南 (Linux 2核2G)

本指南将指导你将 `UnionServiceApp` (UniApp H5前端 + Node.js后端) 部署到 Google Cloud Platform (GCP) 的 Linux 服务器上。

## 1. 环境准备 (服务器端)

登录到你的 GCP Linux 服务器 (推荐 Ubuntu 20.04/22.04 LTS)，执行以下命令安装必要软件。

### 1.1 更新系统与安装工具
```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y nginx git unzip curl
```

### 1.2 安装 Node.js (v18 LTS)
```bash
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs
```

### 1.3 安装 PM2 (进程管理器)
用于保持后端服务在后台常驻运行。
```bash
sudo npm install -g pm2
```

---

## 2. 项目构建 (本地电脑)

为了节省服务器资源，建议在本地电脑完成前端构建。

### 2.1 编译 H5 前端
在本地项目目录下打开终端：
```bash
cd client
npm install
npm run build:h5
```
*构建完成后，生成的文件通常位于：`client/unpackage/dist/build/h5`*

---

## 3. 上传文件 (本地 -> 服务器)

你需要将**编译后的前端文件**和**后端源码**上传到服务器。推荐使用 SFTP 工具 (如 FileZilla, WinSCP) 或 SCP 命令。

假设服务器目录结构规划如下：
- 前端静态文件: `/var/www/union-h5`
- 后端服务代码: `/var/www/union-server`

### 3.1 准备目录 (服务器端)
```bash
sudo mkdir -p /var/www/union-h5
sudo mkdir -p /var/www/union-server
# 修改权限以便上传 (将 current_user 替换为你的服务器用户名)
sudo chown -R $USER:$USER /var/www
```

### 3.2 上传操作
1.  **前端**: 将本地 `client/unpackage/dist/build/h5/` 文件夹内的**所有内容**上传到服务器的 `/var/www/union-h5/`。
2.  **后端**: 将本地 `server/` 文件夹内的代码上传到服务器的 `/var/www/union-server/`。
    *   *注意：不要上传 `node_modules` 文件夹，我们将在服务器上重新安装依赖。*

---

## 4. 启动后端服务 (服务器端)

```bash
cd /var/www/union-server

# 安装依赖
npm install

# 使用 PM2 启动服务 (假设入口文件是 app.js，如果是 bin/www 请相应修改)
pm2 start app.js --name "union-api"

# 保存当前进程列表，开机自启
pm2 save
pm2 startup
```

*记下后端运行的端口号，通常是 3000。如果不同，请在下方 Nginx 配置中修改。*

---

## 5. 配置 Nginx 反向代理 (服务器端)

Nginx 将负责托管前端页面，并将 API 请求转发给后端。

### 5.1 创建配置文件
```bash
sudo nano /etc/nginx/sites-available/union-app
```

### 5.2 粘贴以下配置
```nginx
server {
    listen 80;
    server_name _;  # 如果有域名，将 _ 替换为域名 (如 example.com)

    # 前端 H5 静态文件
    location / {
        root /var/www/union-h5;
        index index.html;
        try_files $uri $uri/ /index.html; # 解决 history 模式刷新 404 问题
    }

    # 后端 API 反向代理
    location /api {
        proxy_pass http://127.0.0.1:3000; # 假设后端运行在 3000 端口
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```
*按 `Ctrl+O` 保存，`Ctrl+X` 退出。*

### 5.3 启用配置并重启 Nginx
```bash
# 建立软链接
sudo ln -s /etc/nginx/sites-available/union-app /etc/nginx/sites-enabled/

# 检查配置语法
sudo nginx -t

# 如果显示 successful，重启 Nginx
sudo systemctl restart nginx
```

## 6. 配置谷歌云防火墙 (GCP Console)

这是最容易被忽略的一步。默认情况下，GCP 实例可能只开放了 SSH (22) 端口。

1.  登录 **Google Cloud Console**。
2.  进入 **VPC Network (VPC 网络)** -> **Firewall (防火墙)**。
3.  点击 **Create Firewall Rule (创建防火墙规则)** (或者检查是否有 `default-allow-http` 规则)。
4.  设置如下：
    *   **Name**: `allow-http-80`
    *   **Targets**: `All instances in the network` (或指定你的实例标签)
    *   **Source IP ranges**: `0.0.0.0/0` (允许所有来源)
    *   **Protocols and ports**: 勾选 `tcp`，输入 `80`。
5.  保存规则。

---

## 7. 验证部署

打开浏览器，访问：`http://<你的服务器外部IP>`

*   **首页**: 应该能看到你复刻的 UniApp 界面。
*   **API**: 如果页面有网络请求，确保它们是发往 `/api/...` 的相对路径，或者 Nginx 代理正常工作。

---

## 8. 常用维护命令

*   **查看后端日志**: `pm2 logs union-api`
*   **重启后端**: `pm2 restart union-api`
*   **重启 Nginx**: `sudo systemctl restart nginx`
